# 第一周 习题

## 1-1. 为什么要在计算机中配备操作系统?

操作系统是计算机系统中的核心软件, 它充当了用户与计算机硬件之间的桥梁, 提供了一个抽象层来管理和协调计算机资源.

对用户而言, 操作系统提供了一个友好的界面, 使得用户能够方便地与计算机进行交互. 操作系统提供了文件系统来管理数据存储, 提供了图形用户界面 (GUI) 或命令行界面 (CLI) 来简化用户操作, 以及各种工具和应用程序来满足用户的需求. 对于开发者而言, 操作系统提供了丰富的 API 和库函数, 使得开发者能够方便地访问计算机资源, 如文件系统、网络、进程管理等. 操作系统还提供了安全机制来保护用户数据和系统资源, 并提供了多任务处理能力来支持同时运行多个应用程序.

在资源管理方面, 操作系统负责管理计算机的各种硬件资源, 包括 CPU, 内存, 存储设备, 网络接口等. 操作系统通过调度算法来分配 CPU 给不同的进程, 通过内存管理来分配和回收内存资源, 通过文件系统来管理数据存储, 以及通过网络协议栈来管理网络通信. 这不仅提高了计算机的效率和性能, 还确保了系统的稳定性和安全性.

操作系统的存在使得计算机能够更高效地运行和管理资源, 提供了一个稳定和一致的环境来满足用户和开发者的需求, 为计算机的广泛应用提供了基础的软件支持.


## 1-3. UNIX 和 Linux 有什么关系?

UNIX 操作系统是由 AT&T 贝尔实验室在 1970 年代开发的一个多用户多任务的操作系统. UNIX 奠定了现代操作系统的基本形态, 包括文件系统, 进程管理, 内存管理等核心概念, 并发扬了可移植性和模块化设计, 以及一切皆文件的设计哲学.

现代各大主流操作系统如 macOS, BSD 家族, Linux 等都受到了 UNIX 的深远影响: 现代 macOS 是基于 Darwin 内核和 BSD 软件的操作系统, 是目前主流操作系统中唯一通过 UNIX 认证的系统; 现代开源 BSD 家族（如 FreeBSD, OpenBSD, NetBSD等项目）则直接源自 BSD 系 UNIX, 因其稳定性和安全性而享有盛誉; Linux 则是由 Linus 于 1990 年代开始主导开发的一个类 UNIX 操作系统内核, 配合 GNU 项目, 形成了一个完整的操作系统, 在此基础上发展出了众多发行版, 如 EDA 工具支持最好的 RHEL 系和 SUSE 系等企业级系统, 以及广受欢迎的 Debian 系和 Arch 系等社区驱动的系统, 此外 Android 也基于 Linux 内核开发.

Linux 没有继承 UNIX 的任何一行代码, 但它的设计深受 UNIX 的启发, 并以其开源、免费、灵活以及广泛的硬件兼容性和强大的社区支持而闻名. 它与 UNIX 最核心的区别在于它只是一个内核, 而 UNIX 是一个完整的操作系统. Linux 系统中使用的软件主要由 GNU 等开源项目提供, 虽然与 UNIX 软件高度兼容, 但它们在代码实现上完全独立, 功能上也有一些差异. 许多常用命令如 `ls`, `grep`, `sed`, `tar` 等在 Linux 中可能有着与 UNIX 中不同的功能扩展.

传统 UNIX 大多由商业公司开发和维护, 通常只支持特定的硬件平台, 并针对这些平台深入优化, 具有极高的稳定性和安全性. Linux 则由开源社区开发和维护, 支持广泛的硬件平台, 包括个人计算机、服务器、嵌入式设备等, 具有极高的灵活性和可定制性, 但稳定性和安全性略逊于 UNIX.

Linux 高度继承了 UNIX 的设计哲学, 但更体现出其现代性和开放性. 例如, 现代 Linux 普遍采用 `systemd` 等工具作为系统和服务管理器, 省去了传统 UNIX 系统中复杂的手动配置; 传统 UNIX 系统中出于节省资源的考虑, 某些设置只以 0, 1 的形式占用一位, 而 Linux 完全采用更加用户友好的文本配置文件; Linux 还引入了 `/proc` 等文件系统, 提供了更丰富的系统信息和控制接口.

Linux 各大发行版已经成为服务器、嵌入式系统、移动设备以及个人计算机等多个领域的主流操作系统. UNIX 和 Linux 之间的关系可以看作, 前者为后者奠定了设计理念和核心概念的基础, 后者则在此基础上发展出一个独立且广泛使用的操作系统生态. 它们共享着独特的设计哲学和高度兼容的软件接口, 是现代计算机操作系统发展史上的两大重要里程碑.


## 1-4. 在作业处理操作系统中，一个作业从进入系统到退出系统，可分为哪些阶段?

在作业处理操作系统中，一个作业从进入系统到退出系统通常可以分为提交 (Submission), 后备 (Ready), 运行 (Running) 和完成 (Completion) 四个主要阶段.


## 1-5. 在作业处理操作系统中，基本模式的进程，可处于哪些状态?

在作业处理操作系统中，基本模式的进程通常可以处于就绪 (Ready), 执行 (Executive), 封锁 (Blocked) 三种状态.


## 1-6. 什么是作业，什么是进程，什么是线程?

作业 (Job) 是用户设计的可在计算机中运行的程序，一段源程序经编译后成为作业. 它是用户提交给操作系统的一项任务, 通常包含一个或多个程序以及相关的输入和输出.

进程 (Process) 是操作系统中正在执行的一个程序实例, 包含了程序代码, 数据, 以及与执行相关的状态信息. 进程可以创建子进程, 形成一个进程树, 并且可以通过进程间通信 (IPC) 来进行数据交换和协作.

线程 (Thread) 是进程中的一个执行单元, 也被称为轻型进程. 线程共享同一进程的地址空间和系统资源, 但每个线程有独立的状态.
